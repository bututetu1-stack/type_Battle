# TYPE ROYALE（仮称 TypeRoyale）

テトリス99型の「つぶし合い」をタイピングで実現するオンライン対戦ゲーム。
速く正確に打つほど相手に「おじゃま単語」を送り込み、バックログ上限を超えた者から脱落、最後の1人が勝利する。

**Phase 0（ソロ基盤）** に加え、**Phase 1（オンライン接続）** まで実装済み。匿名認証 + Firebase Realtime Database でルームを作成/参加し、共通シードと `startAt` による同期スタート、全プレイヤーのミニボード一覧をリアルタイム表示します。

> 詳細仕様は [`docs/spec-v0.2.md`](docs/spec-v0.2.md) を参照。

## 開発

```bash
npm install      # 依存インストール
npm run dev      # 開発サーバ起動（http://localhost:5173）
npm run build    # 型チェック + 本番ビルド（dist/）
npm run preview  # ビルド結果をローカルプレビュー
npm run lint     # 型チェックのみ
```

## 遊び方（Phase 0）

1. `SPACE` キーでゲーム開始。
2. 画面中央のお題（ひらがな）を **ローマ字** で打鍵。下段にローマ字ガイドが表示される。
3. 一定間隔でお題が自動供給され、間隔は時間経過で短くなる（加速）。
4. バックログ（処理待ちの山）が上限（12）を超えると **TOP OUT（脱落）**。
5. ゲームオーバー後は `SPACE` でリトライ。

ローマ字は複数の打ち方を許容（例: し＝`shi`/`si`、じゃ＝`ja`/`jya`/`zya`、促音っ＝次の子音を重ねる など）。

### バトル要素（ソロでも体感できる先取り実装）

- **連鎖（コンボ）**: 単語をノーミスで打ち切るたびに +1。途中でミスするとリセット。
- **マイルストーン攻撃**: 5・10・15・20… 連鎖の節目で、ランダムな敵（ダミー）へおじゃまを送る（5連鎖ごとに送る数 +1）。敵のバックログが上限を超えると **撃墜（K.O.）** → 自分のバッジが増える。
- **アイテム**: お宝単語（🟨）を打ち切ると抽選で 1 つ獲得。`Enter` で使用。
  - 🛡 **シールド**: 次の自動供給を 1 回無効化
  - 🌀 **おじゃま一掃**: バックログ中のおじゃま単語を消す
  - ⏸ **ブレーキ**: 自動供給を 5 秒停止
- **スコア / 最高連鎖 / SEED**: 結果画面に表示。お題列は SEED から決定論的に生成される（仕様 §4.3 の土台。将来のオンライン対戦で全員同じ初期お題列を共有するための仕組み）。

> 周囲のミニボード（ダミープレイヤー20人）の挙動と「攻撃で撃墜」はソロ用の擬似演出です。実際の対人同期はオンライン対戦で行います。

## オンライン対戦（Phase 1）

ホーム画面で **「オンライン対戦」** を選択 → ロビーで名前を入れて **ルーム作成** か **ルームコードで参加**。

1. **ロビー/ルーム**: 作成すると 4 文字のルームコードが発行される。友達に共有して同じルームへ。
2. **同期スタート**: ホストが「ゲーム開始」を押すと、サーバ時刻基準の `startAt`（5秒カウントダウン）を全員で共有して同時スタート。
3. **対戦**: 各自がローカルで打鍵し、`backlog / combo / kpm / alive` のサマリのみを ~400ms ごとに書き込み。全員のミニボードがリアルタイム更新される。
4. **決着**: トップアウトすると順位確定 → 観戦。生存 1 人で結果画面（順位・KPM）。

> お題列は共通シードから各クライアントがローカル生成（キーストロークは同期しない＝低トラフィック / 仕様 §6.3）。

### 攻撃・相殺（Phase 2）

オンライン対戦では「つぶし合い」が成立します。

- **マイルストーン攻撃**: 5・10・15… 連鎖の節目で、生存中のランダムな相手におじゃまを送る（`/attacks/{targetUid}` に push）。
- **予告ゲージ**: 受けた攻撃は即着弾せず、画面左の赤ゲージに溜まり **1.5 秒後に確定** してバックログ末尾におじゃま単語として挿入される。
- **相殺**: 確定前に自分が攻撃を出すと、受信予告と **1:1 で相殺**。相殺しきれなかった余剰だけが相手へ飛ぶ（受けてから打ち返して帳消しにする攻防 / 仕様 §3.3）。
- **ピンチ時倍率**: 自分のバックログが上限の 70% 以上だと攻撃 ×1.5（瀕死から一発逆転 / 仕様 §3.6 B）。

> ターゲティング4モード・バッジ/KOボーナスは Phase 3 で実装予定（現在は常にランダムターゲット）。

### Firebase 側の設定（公開前に必要）

このアプリは `type-283c2` プロジェクトに接続します。Firebase コンソールで以下を有効化してください。

1. **Authentication → Sign-in method → 匿名（Anonymous）を有効化**（未設定だとサインインに失敗します）。
2. **Realtime Database → ルール** に [`database.rules.json`](database.rules.json) の内容を反映（認証済みユーザーのみ読み書き可、`players/{uid}` は本人のみ書込）。

> Web 用 `apiKey` はクライアント公開前提の値で秘密ではありません。アクセス制御はあくまで上記の DB ルールで担保します。
> Spark（無料）プランは Realtime DB の **同時接続 100** が上限。まずは 1 ルーム 8〜16 人想定です（仕様 §6.5）。

## GitHub Pages デプロイ

`main` ブランチへの push で `.github/workflows/deploy.yml` が走り、`dist/` を GitHub Pages に公開します。
リポジトリの **Settings → Pages → Build and deployment → Source** を **「GitHub Actions」** に設定してください。
公開 URL: `https://<owner>.github.io/type_Battle/`

ビルド時の `base` パスは `/type_Battle/` に設定済み（`vite.config.ts`）。

## ディレクトリ構成

```
src/
  App.tsx              ルーター（ホーム / ソロ / ロビー / ルーム）+ 匿名認証

  index.css            Tailwind 読み込み
  components/
    SoloGame.tsx       ソロ練習（お題・判定・連鎖・攻撃・アイテム）

    OnlineRoom.tsx     ルーム購読 + 待機画面 + presence
    OnlineGame.tsx     オンライン対戦本体（同期スタート・サマリ書込・観戦・結果）
    MiniBoard.tsx      ミニボード（ソロのダミー / オンラインの他プレイヤー共通）
  lib/
    rng.ts             決定論 PRNG（mulberry32）
    words.ts           お題プール・ローマ字マップ・トークナイザ・単語生成
    engine.ts          打鍵判定コア（純粋関数 processKey）
    types.ts           共有型定義
    firebase.ts        Firebase 初期化 + 匿名認証
    room.ts            ルーム CRUD / presence / サーバ時刻同期 / 攻撃送受信
docs/
  spec-v0.2.md         仕様書
database.rules.json    Realtime Database セキュリティルール
```
